High-performance Flask-compatible framework and WSGI server as drop in replacement for Gunicorn + Flask in a single package.
# still unreleased... pip install cruet
python -m cruet run your_module:app

# before
from flask import Flask

# after
from cruet import Flask

Improvement vs Flask + Gunicorn

Why Cruet

Full compatibility

No rewrite required. Just load Cruet instead of Flask.

Speeeeed!

Routing, HTTP parsing, cookies, and response formatting all in C.

Independence

Standalone WSGI pre-fork application server.

Performance

Hello world ~41× throughput
Routing-heavy endpoint ~24×
Middleware chain ~33×
JSON endpoint ~9.7×
(With still plenty of room for additional optimization)

What's in the box

Optional async server via libevent

Event loop with multi-worker pre-fork model.

Safe Sync Fallback

Build succeeds even without libevent.

Flexible Deployment

Supports TCP and UNIX sockets.

Flask compatible syntax

Flask 3.0+, as you use it, all supported
